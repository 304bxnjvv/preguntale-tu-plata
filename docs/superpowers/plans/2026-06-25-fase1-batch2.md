# Fase 1 producto — Batch 2: onboarding WOW + privacidad + cuenta (Plan 9, autónomo)

**Goal:** Primera impresión que genera WOW antes de pedir esfuerzo (patrón Rocket Money) + privacidad real (no solo eslogan) + cumplir requisito de borrado de datos. Construido con defaults; dudas anotadas para el usuario.

## Contrato API (nuevo)
- `POST /demo/seed` → 201 `{ "inserted": n }` — inserta ~18 transacciones chilenas realistas (fuente="demo") repartidas en mes actual + anterior. Idempotente (no duplica si ya hay demo).
- `DELETE /demo/seed` → 200 `{ "deleted": n }` — borra solo las transacciones fuente="demo".
- `DELETE /account/data` → 200 `{ "transactions": n, "chat": n, "uploads": n }` — borra TODOS los datos del usuario (transactions, chat_messages, uploads). NO borra la cuenta Auth (eso requiere service-role; queda como duda).

### Task 1 — Backend
- `app/services/demo_service.py`: `seed_demo(session, user_id)` (18 txns chilenas: sueldo ingreso, Líder, Jumbo, Rappi x3, Uber x2, Copec, Netflix, Spotify, Cruz Verde, Falabella, etc., con fechas en mes actual y anterior, categorías ya pobladas, fuente="demo"); `clear_demo(session, user_id)`. Idempotente.
- `app/services/account_service.py`: `delete_user_data(session, user_id)` → borra transactions + chat_messages + uploads del user; devuelve conteos.
- `app/services/extraction_service.py`: **enmascarar RUT y N° de cuenta antes de mandar a OpenAI** en el path de texto (PDF/CSV): regex RUT chileno (`\d{1,2}\.?\d{3}\.?\d{3}-[\dkK]`) y secuencias largas de dígitos (≥10) → reemplazar por `[RUT]`/`[CUENTA]` en el texto antes de `_extractor().invoke`. NO afecta la extracción de transacciones (montos/fechas/glosas se mantienen).
- `app/api/routes/`: endpoints arriba, router registrado en main con prefijo `/api/v1`.
- Tests: demo seed idempotente + categorías; clear demo; delete_user_data borra las 3 tablas y aísla por user; masking tacha RUT/cuenta y deja intactos montos/fechas.

### Task 2 — Frontend
- **OnboardingScreen** (`/onboarding`): se muestra a usuarios nuevos (0 transacciones). Pasos: (1) bienvenida + **privacidad** ("no le pedimos permiso a tu banco, solo los archivos que tú subes; tus datos se borran a los 30 días o cuando quieras"); (2) **pregunta estimativa** ("¿cuánto crees que gastas al mes en delivery?") con input; (3) elegir **"subir mi cartola"** (→ /upload) o **"probar con ejemplo"** (→ POST /demo/seed); (4) **reveal**: el número real (gasto en Comida y delivery del mes, desde summary) vs su estimación + CTA "pregúntale a tu plata" (→ /chat). Skippable ("saltar").
- Trigger: en el dashboard, si `transactionsProvider` está vacío y no se ha completado onboarding, navegar a `/onboarding` (o mostrar botón "Empezar"). Mantener el empty state actual como fallback.
- **api_service**: `seedDemo()`, `clearDemo()`, `deleteAccountData()`.
- **Perfil/Ajustes**: un acceso (ícono en AppBar del dashboard o pantalla simple) con **"Eliminar mis datos"** (confirm dialog → DELETE /account/data → signOut) y un texto de privacidad. Reutiliza el logout existente.
- Mensajería de privacidad visible también en la pantalla de upload.
- Tests + analyze verdes; no romper diseño.
