# Fase 1 producto — Batch 1: suscripciones + chat vivo (Plan 8)

**Goal:** Hacer el valor PAGABLE evidente: detectar suscripciones recurrentes (el "número que duele") y que el chat responda comparativos mes-a-mes con datos reales (tesis "tu banco te muestra, esta te explica").

> #2 del plan GTM (diccionario comercios chilenos) YA está hecho en Plan 7 (`categorias.py`).

## Contrato de API (nuevo)
- `GET /insights/suscripciones` → `{ total_mensual: number, items: [{descripcion, monto, categoria}] }`
- `GET /insights/comparativo` → `{ mes_actual: str, mes_anterior: str, gastos_actual: number, gastos_anterior: number, delta: number, top_cambios: [{categoria, delta}] }`

### Task 1 — Backend (insights + chat con persona)
- `app/services/insights_service.py`:
  - `detectar_suscripciones(session, user_id) -> dict`: candidatos = transacciones gasto (monto<0) con `categoria == "Suscripciones"` UNION las que se repiten (misma descripcion normalizada, monto similar ±10%) en ≥2 meses distintos. Devuelve items (dedup por descripcion, último monto) + `total_mensual` (suma estimada mensual).
  - `comparativo_mensual(session, user_id) -> dict`: gasto total (monto<0) del mes calendario actual vs el anterior, delta, y top 3 categorías con mayor cambio.
- `app/api/routes/insights.py`: endpoints GET de arriba (auth + session). Registrar el router en la app.
- `app/rag/rag_service.py`: PROMPT con **persona**: asistente cálido chileno (tono cercano, minúscula, sin juicio, "plata" no "dinero"), proactivo y específico; instruir a SUMAR paso a paso (mantener). En `ask()`, inyectar al contexto un bloque "Resumen del usuario" con el comparativo mensual + suscripciones detectadas, para que responda "¿cómo voy vs el mes pasado?" / "¿cuánto gasto en suscripciones?" con datos reales (cargar vía insights_service).
- Tests: detección (categoria + recurrencia), comparativo (excluye otros meses), endpoints 200 con auth, el contexto del chat incluye el resumen.

### Task 2 — Frontend (card suscripciones + línea comparativo)
- `api_service.dart`: `getSuscripciones()` y `getComparativo()`.
- `models/`: `Suscripciones {total, items}`, `Comparativo {...}`.
- `data_providers.dart`: `suscripcionesProvider`, `comparativoProvider`.
- `dashboard_screen.dart`: card glass **"Suscripciones detectadas — $X/mes"** con lista (ícono, estilo theme) + una línea sutil bajo los montos: "vs mes pasado: ↑/↓ $Y" (color salvia/salmón). Respetan el diseño actual.
